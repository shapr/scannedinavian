* Alacrity
Brian Warner has this idea of "alacrity".
He uses it to describe how much encrypted data and request/responses must be accumulated to get the first few decrypted plaintext bytes.

We took that idea to mean explicit per block control would be good.
We wanted to be sure that we could cryde (crypt+decrypt) files larger than the amount of memory.
We came up with the idea of "explicit heap-bounded work".
Our code in magic-cap does block by block work.
